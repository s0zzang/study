섹션 0. 자바스크립트의 개요와 개발환경 구축

1) 웹 프로그래밍 및 자바스크립트 개요 (22.02.13)

  • 웹 프로그래밍이란?
    • 백엔드 프로그래밍

      : 서버에서 데이터 관리를 프로그래밍 (웹서버, DB, PHP, JAVA/JSP, ASP)

    • 프론트엔드 프로그래밍

      : 서버에서 받아온 정보를 웹 브라우저에 어떻게 표시할 것인지 프로그래밍

  • 자바스크립트의 역할
    • 웹사이트를 동적으로 만들 수 있다
    • 웹 브라우저에서 실행되는 프로그램을 만들 수 있다
    • 서버를 구성하고 서버용 프로그램을 만들 수 있다
      • node.js : 서버를 만들 때 사용하는 자바스크립트 프레임워크
  • 자바스크립트의 특징
    • 모든 웹 브라우저에서 작동
    • 풀스택 웹 개발 뿐 아니라 다양한 용도의 프로그램을 만들 수 있음
    • 다양한 자바스크립트 공개 API를 사용할 수 있음
      • Application Programming Interface

        : 프로그램들이 서로 상호 작용하는 것을 도와주는 매개체

        : 일기예보 > 지도 > 네이버 / 다음 / 구글에서 제공하는 지도 API 활용

    • 다양한 라이브러리와 프레임워크를 사용할 수 있음
      • 라이브러리

        : 필수 기능 / 자주 사용되는 기능만 가져다 쓰는 것

        : 소프트웨어를 개발하기 쉽게 어떤 기능을 제공하는 도구

      • 프레임워크

        : 이미 준비된 일종의 틀 / 뷰, 리액트 등등

        : 특정 프로그램을 개발하기 위한 여러 요소들과 메뉴얼인 룰을 제공하는 프로그램

섹션 1. 변수와 자료형 그리고 연산자

변수와 기본 자료형 (22.02.19)

  • 변수 : 변할 수 있는 값
    • 변수 선언 규칙
      • 의미 있는 이름 짓기
      • 문자/밑줄/달러로 시작
    • 변수 선언 / 값, 식 할당
      • 권장 : 선언과 동시에 초기화
    • function calc( this! ) < 매개변수, 파라메터, 인자값, 아규먼츠
  • 자료형 : 데이터의 타입
    • 기본형 : 숫자 / 문자열 / 논리형 / undefined / null
      • 숫자 : 정수(소수점X) / 실수(소수점O)

        ★정밀한 실수 계산 못함★

        0.1 + 0.2 = 0.30000000000000004

      • 논리형 : true / false
      • undefined : 변수 선언하고 정의하지 않은 상태
      • null : 값이 유효하지 않은 유형
    • 복합형 : 배열 / 객체
      • 배열 : 하나의 변수에 여러 값 저장하는 유형
      • 객체 : 함수와 속성이 함께 포함된 유형 (console로 요소 찍었을 때)
    • [typeof] 를 통해 타입 분별 가능
  • 연산자 : 사칙 / 논리

배열, 객체, 자바스크립트 자료형의 특징 및 연산자 (22.02.22)

  • 배열
    • 하나의 변수에 여러 값 저장
    • 메모리가 연결되어 있음 (엄밀히 말하면 주소)
    • 예시 [ 사과, 배, 딸기 ]
  • 객체
    • 여러 자료를 중괄호 { } 로 묶은 것
    • 키, 값을 한 쌍으로 여러 자료 저장
    • 예시 var kim = { firstName : ‘ho’, lastName : ‘kim’, age: 35 }
  • 자바스크립트 자료형의 특징
    • 느슨한 자료형 체크 : 미리 변수의 자료형을 지정하지 않음

      = int, double(자바 자료형)을 미리 지정하지 않고 var로 변수 선언

    • 변수를 지정하고 원하는 값을 할당만 하면 됨
    • 타입(자료형)을 어마무시하게 체크하는 언어 > 타입스크립트
  • 연산자
    • 산술 연산자 : 사칙, 나머지(%), 증감(++, —)
    • 할당 연산자 (복합대입 연산자)
      • =, -=, *=, /=, %=
      • 값/문자열 누적 시, 반복문에서 많이 사용
    • 연결 연산자
      • 문자열과 문자열 연결
      • + 기호 사용
    • 형변환
      • 숫자형과 문자형을 더하면 숫자를 문자열로 인식함
      • 곱하기, 나누기, 나머지 연산에서 문자형 자료를 모두 숫자로 자동 인식

섹션 2. 조건문 그리고 반복문

조건문 : IF문, 조건 연산자, switch문 (22.03.09)

  • IF문 (사용 범위가 넓음)
    • 조건연산자 (삼항 연산자)
      • 조건이 하나이고 실행할 명령도 하나일 때 사용
      • (조건) ? 명령1 : 명령2
        • true인 경우 명령1 / false인 경우 명령2
        • 예시) (score ≥ 60) ? alert(’통과’) : alert(’실패’)
    • 중첩 if문
      • if 안에 if문 사용
  • switch문 (한정된 데이터 / 가독성이 높음)
    • 모든 switch문은 if문으로 변경이 가능함

      (모든 if문은 switch문으로 변경이 불가능함)

    • 정수, string 값만 조건 할당 가능
    • 기본 옵션 : switch > case > break > default
    • 예시) var session = prompt(’1마케팅, 2개발, 3디자인’, ‘1’);

      switch(session) {

      case “1” : console.log(’1번’); break;

      case “2” : console.log(’2번’); break;

      case “3” : console.log(’3번’); break;

      default : alert(’잘못 입력했습니다’)

      }

반복문 : for, while, do ... while (22.03.09)

  • for ( 반복 횟수 기준 )
    • 카운터 변수를 기준으로 명령을 여러 번 실행
    • 여러 명령을 늘어 놓지 않고 소스를 간단하게 작성할 수 있음
    • 소스의 양이 적어 실행 속도가 빨라짐
    • for ( var i = 1; i < 6; i++ ) { sum +=i; }
      • 초기화 : var i = 1;
      • 조건 : i < 6;
      • 실행문 : sum += i; (= sum = sum + i)
      • 증감 : i++
  • while ( 특정 조건에 따라 반복하는 경우 )
    • 무한 루프 전용 ( 게임, 자판기, 장비 등 )
    • 빠져나갈 구멍( break문 ) 필요
  • do while ( 특정 조건에 따라 반복하는 경우 )
    • 사용자에게 일단 물어보는 경우
    • 문장 실행 후 조건 확인
    • 예시) ‘계속 하시겠습니까?’ 물어보는 경우
  • break문
    • 반복문의 흐름에서 바로 빠져나올 때 사용
  • continue문
    • 주어진 조건에 맞는 값을 만났을 때 실행하던 반복문을 건너뛰고

      반복문의 맨 앞으로 되돌아감
