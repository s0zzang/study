클래스 작명
클래스 작명에 규칙은 있지만, 정해진 권장 사항은 없음
규칙 : 숫자로 시작 불가, 이모지 불가
너무 컨텐츠에 종속되는 작명은, 추후 컨텐츠 변경시 대략난감 ; ; ;
마진 병합현상 입문
조건: block level element, 인접해있는 상하단 마진만 중첩 (당연함 블럭 요소라 좌우 겹칠 수 없음)
css 제작하며 의도적으로 마진이 겹치도록 설계됨 : 시각적 표현할 때 자연스럽게 보이도록
inline-block에선 적용되지 않음
형제간 마진중첩은 혼란스럽지 않지만, 상하위 요소간 마진겹침은 조금 헷갈릴 수 있음 - 아래부터 상하위 요소간 겹침
.wrap {margin: 50px;} .wrap .box {margin: 20px;}
위의 wrap의 마진과 box의 마진이 병합됨. body > .wrap > .box 인 경우, body의 마진과 box의 마진도 중첩됨
자식의 마진이 위로 돌출되어 부모의 마진과 병합됨
인접해있는 요소들의 마진이 하나로 병합되어 표현됨(body - .wrap 인접, .wrap - .box 인접 == 3가지 모두 인접됨)
상하위 요소의 마진이 중첩되면, 가장 큰 값의 마진이 중첩된 요소의 최상단 마진으로 적용되고(?) 나머지 마진은 0으로 표현됨 (8자 안나오니까)
마진 병합현상 해결방법
1. block이 아닌 inline-block 요소로 활용
2. border/padding을 추가하여 마진이 중첩되는 경계를 분리시킴 (border를 어디에줘야 하는지 테스트 해보기) - 부모자식간 마진 병합만 해결 가능
포인트는 연구가 아닌, 해결 !
3. 부모에 display: flow-root 설정 (root = html의 성질을 가질 수 있도록 설정, block 계열임(flex도 포함)) - 부모자식간 마진 병합만 해결 가능
4. 부모에 overflow scroll, hidden : 스크롤 영역 밖으로 자식이 돌출될 수 없기 때문에 - 많이 사용되는 방법
마진 병합 현상 정리
마진 병합 현상이 생기면, 브라우저는 마진을 모두 0으로 변경, 가장 큰 마진 수치를 찾아서 최상위 요소에 표현
이유는 모름. css 개발자가 알 것 ... (슨생님도 아직 모름)
해결방법: 부모와 자손을 분리하는 컨텐츠 부여 (border, padding, 컨텐츠 등)
html 요소는 마진 병합이 발생하지 않음. 루트(최상위)가 아래로 내려가면 안되기 때문에
실무: 부모 패딩 + 자식 마진탑
