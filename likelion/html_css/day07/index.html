min-width: auto의 의미
flex-basis 설정값보다 이미지(내부 컨텐츠)가 클 때 이미지가 커지면서 부모도 커짐
이유 : flex item은 내부 컨텐츠보다 더 작아질 수 없음 (4번 반복함)
ㄴ min-width : minimum width 최소 너비 | 이니셜 : auto = 내부 컨텐츠 크기 = 이것보다 더 못작아짐
ㄴ 기본으로 설정되어 있음
결국 자식의 컨텐츠 크기가 부모 컨테이닝 블럭 크기를 결정함
min-width:auto = 자식 컨텐츠 크기 = 부모 컨텐츠 크기가 됨 =/= flex-basis
min-width:0 = flex-basis =/= 자식 컨텐츠 크기
즉, flex-basis 설정값을 따라가되, 컨텐츠가 basis로 설정한 값보다 크면 min-width:auto로 인해 내부 컨텐츠보다 작아질 수가 없음
내부 컨텐츠 크기만큼이 아닌, basis 설정값대로 표현되려면 min-width:0 필요
flex-basis:0 > 0 안됨 > min-width: 0 > 추가하면 0 됨
flex-basis vs width
width:200px; 하면 컨텐츠보다 작아짐 - 이유 : width 설정시 min-width까지 설정됨
width도 shrink 영향 받음
basis보다 width 사용하는게 편함
width 때리면 basis도 영향받고 basis 값으로 전달된 것.
width도 역시 flex-grow 적용 받음 - 싫으면 grow:0
flex-direction, flex-wrap
Flex container에 사용
flex-wrap: flex-basis 보장 - 수축할 필요가 없어서 (shrink:1 그대로있음)
justify-content와 align-items
flex-container 에게 주는 값
flex item을 정렬하기 위한 프로퍼티
justify-content
justify-content : text-align 느낌 / flex-start, flex-end, center, ...
align-items
flex item 한개의 높이가 커짐 = 부모 높이 커짐 = 다른 item 높이 커짐
초기값 : normal = stretch
stretch된 요소에 margin: auto 주면 가용 영역이 여백으로 적용됨 (flex container 한테 크기 지정했을 경우)
ㄴ stretch(부모)보다 margin(자식)이 더 우선됨
ㄴ flex item은 상하 마진 auto 도 가능함 !! 이햐 ~
align-content도 있긴 함 ~
사이드바, 컨텐츠 레이아웃 관련
컨텐츠 내부 컨텐츠가 클 경우, shrink:1 > 부모 관점에서 수축되어, 내부 컨텐츠 줄바꿈됨
방법1 - 첫날) 사이드바 : basis 200, shrink 0 | 컨텐츠 : basis auto, shrink 1
방법2 - 오늘) 사이드바 : basis 200, shrink 1 | 컨텐츠 : basis 0, shrink 1, grow: 1
ㄴ basis : 0 - 콘텐츠 너비만큼만 자리 차지하기 때문
ㄴ shrink : 1 - 줄바꿈
ㄴ grow : 1 - 가용여백 몰빵
Main-axis와 Cross-axis
main-axis: 주축 - justify-content
cross-axis: 교차축 - align-items
