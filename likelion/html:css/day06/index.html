Flexbox 동작 방식 이해
정렬을 희망하는 요소의 부모에게 display: flex
부모 : flex Container
자식 : flex items
가로 배치가 되어도, inline / inline-block 요소가 아님 (마진 가능, 띄어쓰기 없음)
block 레벨의 가용 공간을 auto 값으로 활용할 수 있는 것처럼 flex도 하숙생들끼리 가능함
flex items는 인라인 요소라도 block처럼 사용할 수 있음
flex-grow flex-shrink flex-basis(기초) 이해하기
위의 프로퍼티는 flex-item에 사용함
flex-basis (아이템 크기 관련) : 컨텐츠 크기 만큼 / 초기값 - auto
ㄴ flex-basis: 200px > 200px 위드의 상자가 됨
ㄴ flex-basis: 0 > 내부 컨텐츠와 관계없이 n등분 가능
ㄴ auto, fit-contents, px 등 width 처럼 사용 가느
200px 작성할 경우, 컨텐츠가 많아서 200px을 차지하지 못할 경우 안될 경우 줄어듦
flex-grow (아이템 크기 관련) : 할당 가능한 공간의 정도 설정, 여백 없이 방 사이즈 증가 / 초기값 - 0
flex-grow: 1; = 1:1:1 > 비율을 의미함
ㄴ flex-item에 직접 작성하면 0:1:0 으로 남은 여백이 모두 grow 적용한 요소에 방사이즈로 편입됨
이때 비율은, 컨텐츠 크기 + (가용공간/n) 으로 증가함 == 무조건 정확한 n등분이 아님
비율 설정시 (예시 1:2:3)인 경우 각각의 flex-item에 flex-grow:1; flex-grow:2; flex-grow:3; 부여
두개 같이 활용하기
flex-basis: 200px, 2번째 요소만 flex-grow: 1;
ㄴ 양옆 상자는 200px 고정, 2번째 요소는 남은 가용 여백 모두 차지
1 2 3
두개 활용하는 경우, 200px & calc(100% - 400px) 보다 정확하게 200px을 차지함
flex-shrink (수축) : 수축이 되는 비율 / 초기값 : 1
basis값이 언제든 보장되지 않고 더 작아질 수 있음. 그때 작아지는 비율 관련 설정
flex-item은 내부 컨텐츠보다 작아질 수 없음. Container을 뚫고 나감
flex-shrink: 0 = 난 수축 안하겠다 이거야 난 절대 안줄어들ㅇㅓ ~ > basis 보장
ㄴ 나머지 다른 요소가 줄어들어야 하는 크기를 균등 분배
