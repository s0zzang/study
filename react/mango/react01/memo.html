node_modules

  • 모듈이 모여서 리액트 문법으로 작성된 코드를 자스로 변환시켜줌
  • node를 관리하는 프로그램은 npm

src

  • 소스 폴더
  • html 파일이 하나도 없음 - js도 html을 기반으로 해석됨
  • src 내용을 컴파일 하여 public 폴더 내에 html 생성

public

  • node modules 내에 있는 바벨이 src 내에 있는 파일을 public 내 파일로 변환
  • public 파일을 수정하는 것이 아님 !!

package.json

  • dependencies : 의존성 - 라이브러리/플러그인 버전 관리
  • scripts : 명령어
  • package.json에 작성하면 node modules를 통해 촤작 세팅해줌

필요한 파일

  • src/App.css
  • src/App.js
  • src/index.css
  • src/App.js
  • 나머지는 테스트를 위한 파일이라 필요 없음
  • index.js 내에서도 역시 삭제

컴포넌트

  • function App에서 div태그를 반환 => App() : 컴포넌트
  • 컴포넌트는 대문자로 만듦
  • index.js : return값 없음
  • 두 번 이상 재사용 되는 것
  • 종류 : 클래스형, 함수형
  • 클래스형 : 16.8 이전
  • 함수형 : 16.8 이후

render

  • react 문법으로 작성된 코드를 변환해서 그리는 함수
  • index.js 에서 root.render로 전달되어 index.html 문서로 그려짐

- 주석 : { /* 주석 */ }

컴포넌트 쪼개기

  • 하나의 컴포넌트 : 하나의 dom 요소
  • 컴포넌트 요소가 div처럼 많아짐
  • 컴포넌트를 어떻게 쪼개고, 네이밍 하는지가 상당히 중요함
  • App: 데이터 공급 담당 (div 태그 없음)
  • ExpenseItem: 목록을 출력하는 단순한 UI, 금액, 타이틀 담당
  • ExpenseDate: 날짜를 출력하는 컴포넌트, app과 ExpenseItem을 거쳐 전달받은 날짜 데이터를 분해하여 출력하는 비교적 복잡한 구조 담당
  • 호출하는 쪽에서 값 전달, 호출 받는 쪽에선 값을 받아야 함
